TSP Visualization

Problem

The Traveling Salesman Problem asks for the shortest route visiting all cities exactly once and returning to the start. It's NP-hard — no polynomial-time algorithm is known for all instances. The Christofides algorithm produces solutions within 1.5× the optimum for metric instances.

Implementation

My modified approach builds a minimum spanning tree (MST), then performs a depth-first search recording visit order. This reduces time complexity from O(n³) to O(n² log n) compared to the full Christofides algorithm.
